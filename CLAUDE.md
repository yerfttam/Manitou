# Manitou

Marketing website for **Manitou Lodge** — a vacation rental property in Forks, Washington offering lodge rooms, cabins, and camping on 10 acres of coastal rainforest. Minutes from Olympic National Park, Rialto Beach, and the Washington coast.

## Project Structure

```
NEW/                    ← source of truth, this is what gets deployed
  index.html            ← Home page (complete)
  accommodations.html   ← Property listings (WCBNW-style: sticky cat nav, carousels, rendered from listings.json)
  about.html            ← About / history / regional info
  policies.html         ← Rates, cancellations, pets, check-in/out
  specials.html         ← Specials (coming soon placeholder)
  contact.html          ← Contact form (Formspree) + info panel + feature strip
  js/nav-loader.js      ← dynamically injects shared nav + dropdown CSS into pages
  js/footer-loader.js   ← dynamically injects shared footer into pages
  js/listings-loader.js ← fetches listings.json, renders category sections + carousels on accommodations.html
  partials/nav.tpl      ← shared nav markup (desktop + mobile, Accommodations dropdown)
  partials/footer.tpl   ← shared footer markup (links, contact, version)
  fonts/                ← Native Record font family (Sans, Script, Serif) woff/woff2
  images/               ← Website assets (backgrounds, property photos, logo, icons)
    logo/               ← MTL-Logo_Light.png (stag badge, used in nav + footer)
    hemlock.jpg         ← Welcome section photo (left)
    manitou-sign.jpg    ← Welcome section photo (right)
    wcb-2.jpg           ← Rainforest section background
    ig-1..4.jpg         ← Instagram strip photos (from saved IG pages)
  data/                 ← JSON data files
    listings.json       ← Guesty property listings (generated by scripts/sync-guesty.py)
scripts/                ← Utility scripts
  sync-guesty.py        ← Fetches listings from Guesty API and writes listings.json
serve.py                ← Local dev server with clean-URL (.html fallback) support — mirrors Render behavior
```

## Deployment

- **Hosting:** Render (static site), auto-deploys on push to `main`
- **Publish directory:** `NEW`
- **Build command:** none (pure HTML/CSS/JS, no build step)
- Config: `render.yaml` in repo root

## Local Dev Server

```bash
python3 serve.py
```

Serves `NEW/` at **http://localhost:3456** with clean-URL support (`.html` fallback), mirroring Render's behavior. `python3 -m http.server` does NOT support clean URLs and will 404 on nav links — always use `serve.py` instead.

**Important:** The preview server (and any server that does clean URL rewrites) will break `fetch()` calls for `.html` files. Partials use `.tpl` extension to avoid this. Nav and footer are always `partials/nav.tpl` and `partials/footer.tpl`.

## Navigation

The nav is a shared partial at `partials/nav.tpl`, injected at runtime by `js/nav-loader.js` into a `<div id="site-nav-placeholder">` in each page.

The footer is a shared partial at `partials/footer.tpl`, injected by `js/footer-loader.js` into `<div id="site-footer-placeholder">`.

Each page must include both:
```html
<div id="site-nav-placeholder"></div>
<script src="js/nav-loader.js" defer></script>
...
<div id="site-footer-placeholder"></div>
<script src="js/footer-loader.js" defer></script>
```

**Footer CSS must be included on every page** — it is not in the partial, it lives in each page's `<style>` block. Copy the `/* ── Footer ── */` block from any existing page when building a new one.

## Design

Visual design is from the designer's PDF: `assets-raw/Manitou Lodge/Website/1 HOME/MTL - HOME PAGE V2.pdf`

To read it (it exceeds Claude's 20MB limit), convert to images first:
```bash
python3 -c "
import fitz
doc = fitz.open('assets-raw/Manitou Lodge/Website/1 HOME/MTL - HOME PAGE V2.pdf')
page = doc[0]
h, w = page.rect.height, page.rect.width
sections = [('hero',0,0.22),('about',0.22,0.42),('spaces',0.42,0.58),('rainforest',0.58,0.72),('rain',0.72,0.90),('footer',0.90,1.0)]
for name, y0, y1 in sections:
    clip = fitz.Rect(0, h*y0, w, h*y1)
    page.get_pixmap(matrix=fitz.Matrix(2.5,2.5), clip=clip).save(f'/tmp/manitou-{name}.jpg')
"
```
Then read `/tmp/manitou-{section}.jpg` files.

### Color Palette (designer spec — all verified)

| Variable | Value | Role |
|----------|-------|------|
| `--green-dark` | `#373f2e` | Primary background (nav, footer, hero, section headers) |
| `--green` | `#3a4a24` | Mid green (secondary accents — avoid for backgrounds) |
| `--amber` | `#c4822a` | CTA buttons, accents, script labels |
| `--olive` | `#8a8153` | Secondary/decorative |
| `--cream-light` | `#e4dfb5` | Page background |
| `--cream` | `#ddd9bc` | Cards, info boxes, muted surfaces |
| `--text-dark` | `#2d2a1e` | Primary body text |
| `--text-mid` | `#4a4535` | Secondary body text |

**Rule:** Use `--green-dark` for all dark green backgrounds (nav, footer, hero, section headers, feature strips). `--green` is only for subtle secondary accents.

### Fonts

Native Record family via `@font-face` woff/woff2 in `fonts/`:
- `Georgia, serif` — default `body` font; all paragraph/body copy
- `NativeRecordSans` — section headings, card titles, labels, buttons, nav links, uppercase UI text
- `NativeRecordScript` — decorative script labels (amber, e.g. "Welcome to", "Always welcome to")

All three `@font-face` declarations must be included in every page's `<style>` block.

**Key rule:** Only the main hero `h1` uses NativeRecordSerif. All other headings use NativeRecordSans.

## Version Number

Every page footer displays a version string via `<span class="site-version">`.

**Rules:**
1. Bump the version in `NEW/partials/footer.tpl` before EVERY commit that gets pushed
2. Bump patch (0.x.x) for routine fixes and small changes; bump minor (0.x.0) for new pages or significant features
3. Update `Current version` below to match
4. Lead the commit message with the version number — this is what shows in the Render deploy dashboard

Current version: **v0.5.2**

**The footer is a shared partial.** To update the version, edit ONE file only:
```
NEW/partials/footer.tpl
```
Change the `<span class="site-version">vX.X.X</span>` value. Do not edit version strings in individual page files.

**Commit message format:** Always lead with the version number:
```
v0.4.7 fix: description of what changed
```
This version prefix appears in the Render deploy dashboard, making it easy to confirm which version is live.

## Guesty Integration

- **Account:** Same account as WCBNW (`app.guesty.com`)
- **Manitou listings:** All nicknames start with `z` — WCBNW listings are filtered out
- **Booking engine:** Newer Next.js standalone app at `manitoulodge.guestybookings.com` (engine ID: `1197`) — **no widget embed needed**, direct per-property URLs only

- **Sync script:** `scripts/sync-guesty.py` — reads `NEW/property-map.txt`, fetches Guesty API, writes `listings.json`
- **Nightly sync:** GitHub Actions workflow (to be configured — copy from WCBNW)

### Property Card Descriptions

Cards use `getListingDesc()` in `listings-loader.js` — extracts the first unique paragraph from Guesty's `description` field, skipping the shared campground opener used by A-Frames and Camping. Strips `***` markers, truncates to ~440 characters (4 lines).
